Part 2: WSL vs Desktop Linux & Step-by-Step Installation Guide

3. Core Differences: WSL (Terminal-Based Linux) vs. Full Desktop Linux

When running Ubuntu via the Windows Command Prompt using WSL (Windows Subsystem for Linux), the user experience is fundamentally different from running a full graphical Ubuntu operating system. The version installed through `cmd` is a Command Line Interface (CLI) environment, which opens as a text-based terminal without a visual desktop, wallpapers, or application icons. This method is incredibly lightweight and resource-efficient because it integrates directly into Windows, allowing you to run Linux utilities concurrently alongside your everyday Windows applications without dragging down your computer's RAM or CPU performance. In contrast, the full desktop version provides a complete Graphical User Interface (GUI), complete with a file explorer, web browsers, and visual window management. While a GUI is more familiar and visually intuitive, running it inside Windows via virtual machines can be highly resource-intensive and often slows down your host computer.

For the purpose of Data Science, programming, and cloud deployment, mastering the terminal-based WSL version is far more beneficial than relying on a graphical desktop. In real-world enterprise environments, production data pipelines and machine learning models are deployed on remote cloud servers that completely lack a graphical user interface to maximize processing power. By learning to navigate, process files, and manage projects strictly through text commands in WSL, you are training yourself to work exactly like industry professionals. This prepares you to seamlessly log into any remote cloud infrastructure, automate data-heavy workflows with scripts, and handle massive datasets directly through the command line without the unnecessary overhead of a graphical desktop.

 4. Step-by-Step Installation Guide for WSL and Ubuntu on Windows

To set up a fully functioning Linux terminal environment directly inside Windows using WSL, follow these precise sequential steps:

Step 1: Check System Requirements. Ensure you are running Windows 10 (Version 2004 and higher) or Windows 11. Your system must support virtualization, which is enabled by default on most modern machines.
Step 2: Open Command Prompt as Administrator:Press the **Windows Key**, type **cmd** or **Command Prompt**, right-click on the application icon, and select .Run as administrator. This grants the terminal necessary permissions to modify system features.
*Step 3: Enable WSL and Install Ubuntu To install the default Linux architecture along with the latest Ubuntu release automatically, type the following command and press **Enter**:
```cmd
wsl --install -d Ubuntu

```


*(Note: If you only execute `wsl --install`, Windows will still default to downloading and configuring the latest Ubuntu distribution).*
* **Step 4: Restart Your Computer** Once the terminal informs you that the installation was successful, close all open applications and **restart your PC** to allow Windows to initialize the hidden virtualization components.
* **Step 5: Initialize Ubuntu and Set Credentials** After your computer reboots, a new Ubuntu terminal window will automatically pop up to complete the configuration. It will prompt you to enter a new **Unix Username** and **Password**. Type your preferred credentials; note that the password characters will not appear on the screen while typing for security reasons—simply type it blindly and press Enter.
* **Step 6: Verify the Installation** To confirm that Ubuntu is properly registered and running inside your system, open a standard Windows Command Prompt (`cmd`) and type the status command:
cmd
wsl --list --verbose


The system should output a list confirming that `Ubuntu` is present, operating in version 2 mode, and ready for use.
